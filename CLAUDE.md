# CLAUDE.md

## 프로젝트 컨텍스트

종목명을 입력하면 애널리스트들이 협업해 리서치를 수행하고, 그 결과를 PPTX 리포트로 내보내는 주식 분석 프로젝트입니다.

- 사용자(나)는 **증권사 리서치 담당**입니다.
- 분석 관점은 **가치투자(Value Investing)** 입니다 — 기업의 내재가치, 펀더멘털, 안전마진을 중심으로 판단합니다.

## 위임 규칙

종목 1건이 들어오면 메인(오케스트레이터)이 아래 애널리스트들에게 작업을 위임하고, 결과를 취합해 PPTX 리포트(5부)로 만듭니다.

### 애널리스트 명단

| 에이전트 | 담당 | 데이터 소스 | 산출물 | 리포트 섹션 |
|----------|------|-------------|--------|-------------|
| `fundamental-analyst` | 재무·실적·공시 | DART OpenAPI | 3개년 재무 요약표 + 코멘트 3줄 | **재무** |
| `market-tech-analyst` | 주가·추세·거래량 | FinanceDataReader | 가격 요약표(이평·52주·거래량) | **차트** |
| `daily-econ-indicators` | 금리·물가·고용·환율 | 매크로 지표 | 거시 브리핑 | **시장 배경** |
| *리스크 애널리스트* | 사업·재무·시장 리스크 | (공석) | — | **리스크** |
| *리드/종합* | 가치투자 결론 종합 | (공석) | — | **종합의견** |

> 공석(리스크·리드)은 충원 전까지 **메인 오케스트레이터가 본 문서 규칙을 지키며 대행**합니다.

### 처리 파이프라인

1. **입력 정규화** — 종목명을 6자리 코드로 확정합니다(모호하면 사용자에게 확인). DART는 8자리 `corp_code`를 쓰므로 `corpCode.xml`로 매핑합니다(예: 삼성전자 005930 → 00126380).
2. **병렬 fan-out** — 서로 독립이므로 **한 번에 동시 위임**합니다:
   - `fundamental-analyst` → 재무
   - `market-tech-analyst` → 차트
   - `daily-econ-indicators` → 해당 섹터 관련 거시 배경
3. **리스크 점검** — 위 3개 결과에서 리스크를 추출합니다(현재 메인 대행).
4. **종합의견** — 가치투자 관점으로 종합합니다(현재 메인 대행). **매수/매도 단정 없이 근거까지만.**
5. **PPTX 출력** — 표지·재무·차트·리스크·종합의견 5부로 생성합니다.

### 위임 원칙

- **병렬은 상황에 맞게 — 무조건 병렬이 아닙니다.** 작업 간 의존성을 먼저 판단해 병렬/순차를 고릅니다.
  - **병렬로** 위임하는 경우: 작업들이 서로 **독립**일 때. 예) 한 종목에 대한 fundamental + tech + econ 분석은 입력(종목코드)만 확정되면 서로를 안 기다리므로 **한 응답에서 동시** 위임합니다.
  - **순차로** 위임하는 경우: **앞 결과가 뒤 작업의 입력**일 때. 예) 종목코드 정규화(corp_code 매핑) → 분석, 또는 3개 분석 결과 → 리스크 → 종합의견은 의존성이 있으므로 순서대로 진행합니다.
  - **단건이거나 범위가 좁으면** 굳이 팬아웃하지 않고 해당 에이전트만 단독 호출합니다.
  - 애매하면 **사용자 의도와 비용**을 보고 판단합니다(전체 리포트=병렬 팬아웃, 한 측면만=단독).
- 각 에이전트는 **자기 데이터 소스만** 사용합니다(fundamental=DART, tech=FDR, econ=매크로). 교차 호출하지 않습니다.
- 에이전트는 **가공된 결과 데이터만 반환**하고, 취합·PPTX 생성은 메인이 합니다.

## 산출물 규칙

- 리포트는 다음 **5부** 구성을 따릅니다:
  1. **표지**
  2. **재무**
  3. **차트**
  4. **리스크**
  5. **종합의견**
- 문체는 **"~입니다" 체**(평서·경어체)로 통일합니다.
- 모든 수치에는 **출처**와 **기준일**을 함께 적습니다.
  - 재무: `(출처: DART, {연도}/{분기})` — 예: `(출처: DART, 2025/연간)`
  - 시세: `(출처: FinanceDataReader, 기준일: YYYY-MM-DD)`

## 가드레일

- **매수·매도 단정 표현 금지** — "사라/팔아라", "매수 추천", 목표가 제시는 하지 않고 **판단 근거까지만** 제시합니다. 특정일 매매 타이밍("내일 사라")도 제시하지 않습니다.
- **출처 없는 수치 금지** — 출처를 확인할 수 없는 숫자는 리포트에 넣지 않습니다. 데이터가 없으면 추정하지 말고 "확인 불가"로 명시합니다.
- 리포트 끝에 **학습용 분석임을 명시**합니다. (투자 권유가 아니며 교육·학습 목적임을 밝힙니다.)

## 환경 / 데이터 노트

- **시크릿**: `.env.local`에 보관하며 커밋 금지(`.gitignore` 적용). DART 키는 `DART_API_KEY`이며, `fundamental-analyst` 호환용으로 동일 값의 `DART_KEY` 별칭도 둡니다.
- **Python HTTPS 인증서**: macOS python.org 파이썬은 시스템 인증서를 못 찾아 SSL 검증에 실패합니다(FDR·DART 호출 깨짐). `certifi`로 컨텍스트를 지정해야 합니다:
  ```python
  import ssl, certifi, functools
  ssl._create_default_https_context = functools.partial(ssl.create_default_context, cafile=certifi.where())
  ```
- **DART corp_code 매핑**: `https://opendart.fss.or.kr/api/corpCode.xml` 로 전체 매핑을 받아 종목코드↔corp_code를 변환합니다.
- **사용 가능 라이브러리**: `FinanceDataReader`, `certifi` 설치 확인됨. `pykrx`는 KRX 로그인 정보가 없어 제한적입니다.
